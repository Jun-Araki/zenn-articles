## はじめに

エンジニアとして働いていると、日々さまざまな情報が手元に溜まっていきます。転職活動の記録、副業の打ち合わせメモ、英語学習のノート、技術のキャッチアップ、確定申告の書類、さらには証明写真や健康診断の結果まで様々です。

これらの情報、あなたはどこに保存していますか？

Google Drive、Dropbox、OneDrive、iCloud、Notion、Evernote……。気がつけば複数のクラウドサービスに情報が分散し、「あのファイルどこに置いたっけ？」と探し回った経験はないでしょうか。

私はこの問題を **GitHub リポジトリ + Claude Code** という組み合わせで解決しました。エンジニアが日常的に使っているツールで、人生に関わるあらゆる情報を一元管理する方法を紹介します。

## なぜ GitHub なのか

### エンジニアにとって最も手馴染みのあるツール

エンジニアであれば、Git と GitHub は毎日のように使っているはずです。新しいツールの使い方を覚える必要がなく、すでに身体に染みついた操作感でファイルを管理できます。

### バージョン管理が自動でついてくる

Markdown ファイルで情報を記録すれば、変更履歴が自動的に残ります。「あのとき何を考えていたか」を `git log` で振り返れるのは、他のクラウドストレージにはない強みです。

### プライベートリポジトリで安心

GitHub のプライベートリポジトリを使えば、個人的な情報も安全に保管できます。二要素認証やSSH鍵と組み合わせれば、クラウドストレージ以上のセキュリティを確保できます。

### どの端末からでもアクセス可能

`git clone` さえすれば、どのPCからでも同じ情報にアクセスできます。新しいマシンへの移行も一瞬です。

## 私のリポジトリ構成

実際に運用している「Life」リポジトリのディレクトリ構成を紹介します。

```
~/ClaudeCode/
├── Life/                        # プライベートリポジトリ（人生の記録）
│   ├── Domestic-employment/     # 国内の就職・転職活動の記録
│   ├── Overseas/                # 海外移住・海外就職の情報
│   ├── Works/                   # 職場での必要書類等
│   ├── Personal-business/       # 個人事業・副業相談の記録
│   ├── Study-English/           # 英語学習
│   ├── Study-technology/        # 技術学習
│   ├── Study-other/             # その他の学習（簿記・宅建など）
│   ├── General-affairs/         # 生活全般
│   ├── Trend-daily/             # 日々のIT・AIトレンド情報
│   └── LinkedIn/                # LinkedIn投稿の原稿・画像
│       ├── Posts/
│       └── Images/
│
└── zenn-articles/               # パブリックリポジトリ（Zenn記事、自動デプロイ）
    └── articles/
```

`Life` はプライベートリポジトリ、`zenn-articles` はパブリックリポジトリとして分離しています。公開・非公開の性質が異なるため、Git リポジトリを分けることでセキュリティと運用のしやすさを両立しています。

仕事のこと、勉強のこと、海外移住の調査、日常の事務手続き、さらにはブログ投稿まで、関連するあらゆる情報がこの構成に集約されています。

## Claude Code との組み合わせが真価を発揮する

GitHub だけでも十分便利ですが、**Claude Code を組み合わせることで、このリポジトリは「ただのファイル置き場」から「自分の人生を振り返り、未来を設計するためのツール」に進化します。**

### ディレクトリを指定するだけで過去を振り返れる

Claude Code にディレクトリパスを伝えるだけで、過去の自分の行動や思考を瞬時に振り返ることができます。

たとえば：

- **「`Overseas/` の内容をまとめて」** と伝えれば、これまでの海外移住に向けた調査・面談・相談の履歴を時系列で整理してくれる
- **「`Domestic-employment/` の転職活動を分析して」** と言えば、過去4回の転職活動でどんな企業に応募し、何を重視していたかの傾向を抽出してくれる
- **「`Study-English/IELTS/` の学習進捗を教えて」** と聞けば、Reading・Listening・Writing・Speaking それぞれの対策状況を把握できる

これは Google Drive や Dropbox では絶対にできないことです。ファイルが Markdown というテキスト形式で保存されているからこそ、AI が内容を読み取り、横断的に分析できるのです。

### 過去の意思決定を未来に活かせる

蓄積されたデータは、単なる記録ではありません。Claude Code を通じて、過去の自分の意思決定パターンを分析し、未来の行動計画に反映することができます。

たとえば、私の場合：

- 過去の転職活動の記録から「自分がどんな環境で力を発揮できるか」を言語化できた
- 海外移住の調査メモを横断的に読み返すことで、「なぜインドを選んだのか」という意思決定の根拠を再確認できた
- 副業・個人事業の相談記録から、自分のキャリアの方向性を客観的に見つめ直せた

**人間は過去の経験を忘れます。でも、GitHub に保存しておけば忘れません。そして Claude Code がそれを読み解いてくれます。**

### 日々の情報収集も自動化できる

Claude Code のスキル機能を活用すれば、日々の情報収集まで自動化できます。私の場合：

- **`trend-daily`**: IT・AI・セキュリティの最新トレンドを毎日収集し、`Trend-daily/` に自動保存
- **`linkedin-post`**: LinkedIn 投稿の英文添削と画像生成を自動実行し、`LinkedIn/` に保存
- **`zenn-article-writer`**: まさにこの記事のように、Zenn記事の作成からGitプッシュまでを一気通貫で実行

スキル定義ファイルに保存先ディレクトリを指定しておくだけで、情報が自動的に正しい場所に蓄積されていきます。

## 分散管理の何が問題だったか

以前の私は、こんな状態でした：

| サービス | 保存していたもの |
|---------|----------------|
| Google Drive | 確定申告の書類、履歴書 |
| Dropbox | 業務委託先との共有ファイル |
| OneDrive | 証明写真、健康診断の結果 |
| Notion | 学習ノート、転職活動の記録 |
| iCloud | その他の雑多なメモ |

5つのサービスにファイルが散らばり、「あのファイルどこだっけ？」問題が日常的に発生していました。さらに：

- **検索性が悪い**: サービスをまたいだ横断検索ができない
- **AI との連携が難しい**: 各サービスのファイルを AI に読み込ませるには手間がかかる
- **コストがかかる**: 複数の有料プランを維持する必要がある
- **データポータビリティが低い**: サービス終了やプラン変更のリスクがある

GitHub + Markdown に一元化することで、これらの問題がすべて解消されました。

## 始め方

### 1. プライベートリポジトリを作成

```bash
mkdir Life && cd Life
git init
git remote add origin git@github.com:YOUR_USERNAME/life.git
```

### 2. ディレクトリ構成を決める

自分の生活に合わせてディレクトリを作成します。最初から完璧を目指す必要はなく、必要に応じて追加・変更していけばOKです。

```bash
mkdir -p Works Study-English Study-technology General-affairs
```

### 3. Markdown で記録を始める

日々の活動や学習の記録を Markdown ファイルで保存していきます。フォーマットにこだわる必要はなく、自分が後から読み返してわかる程度で十分です。

### 4. Claude Code と連携する

Claude Code をインストールし、リポジトリのディレクトリで起動するだけで、蓄積された情報を AI が読み取れるようになります。

```bash
cd ~/Life
claude
```

あとは「○○について過去の記録をまとめて」と伝えるだけです。

## 注意点

- **機密情報の取り扱い**: パスワードやAPIキーは絶対にコミットしない。`.gitignore` と `.env` を適切に設定する
- **パスワード管理は専用ツールで**: パスワードやクレデンシャル情報だけは GitHub に保存せず、[Bitwarden](https://vault.bitwarden.com/) などの専用パスワードマネージャーに任せるべきです。GitHub はあくまで「情報の記録」であり、「認証情報の管理」は専用ツールに分離するのが鉄則です
- **大容量ファイル**: 画像や音声ファイルを大量に保存する場合は Git LFS の利用を検討する
- **プライバシー**: 個人情報を含むファイルはプライベートリポジトリに限定し、共有設定に注意する
- **定期的なバックアップ**: GitHub 自体の障害に備えて、ローカルの複数端末に `git clone` しておくと安心です。Git の分散型という特性上、各クローンが完全なバックアップになります

## まとめ

エンジニアであれば、GitHub と Git は毎日使い慣れたツールです。このツールを仕事だけでなく「人生の記録」にも活用することで、情報の分散管理から解放され、過去の自分と対話しながら未来を設計できるようになります。

Claude Code との組み合わせにより、ディレクトリを指定するだけで過去の行動を振り返り、そこから得られる洞察を未来の意思決定に活かすことができます。これは、単なるファイル管理の話ではなく、**自分の人生をバージョン管理する**という発想の転換です。

まずは一つのディレクトリから始めてみてください。気がつけば、あなたの Life リポジトリが最も頼れるナレッジベースになっているはずです。
